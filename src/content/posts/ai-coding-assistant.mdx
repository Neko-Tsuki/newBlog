事情是这样的——前阵子在落落格上加了个"今日推荐"的小卡片，想放在侧边栏随机展示一篇旧文章。原本以为就一个简单的组件，结果折腾了一个周末才弄得像模像样。

怎么说呢，想法很丰满，现实很骨感。

## 一、从哪儿开始的？

起因很简单：某天翻归档，发现自己写过不少文章但好多都被埋在时间线里了。想想也是可惜，写都写了，总得让它们有机会重见天日吧？

最初的设想就三个要求：

1. 从所有**非置顶**文章里随机选一篇
2. 在侧边栏展示标题、摘要和一张小图
3. 每次刷新或者切换页面的时候重新随机

看起来人畜无害对吧？我也是这么想的……

## 二、第一次尝试——硬核纯前端

大概翻了翻Astro的文档，咱的思路是：

> 直接在后端把文章列表传过去，然后在浏览器端用 `Math.random()` 随机选一篇再渲染出来。

说干就干！先搞了个简单的 Svelte 组件：

```svelte
<script lang="ts">
  import { onMount } from "svelte";
  let { posts } = $props();

  let current = $state(null);

  function pickRandom() {
    const candidates = posts.filter((p) => !p.data.pinned);
    current = candidates[Math.floor(Math.random() * candidates.length)];
  }

  onMount(() => pickRandom());
</script>
```

五分钟写完，兴冲冲启动 dev server 一看——嗯？导航切换页面的时候卡片的确刷新了，但有时候会出现**闪一下空白**的情况，观感极其糟糕。

后来排查发现，问题在于页面切换时 Swup 会替换 DOM，组件重新挂载需要时间，而 `onMount` 里的随机逻辑触发时机比 DOM 替换早那么一丢丢——于是出现了一个"数据还没准备好，组件先渲染了"的空窗期。

## 三、换个思路——后端预选多篇

既然客户端随机有延迟问题，那咱就在后端多选几篇丢给前端，然后前端只是"展示"不就好了吗？

说得好听。但这样一来，每次刷新都是同一批候选，如果用户频繁刷新，看到的永远都是那几篇，就失去了"随机"的意义了。

那……再加个基于日期的种子？

```typescript
function seededRandom(seed: number) {
  const x = Math.sin(seed) * 10000;
  return x - Math.floor(x);
}
```

根据日期生成种子，每天换一批候选，这样一天内看到的推荐是固定的，但每天都会更新。听起来好像是个不错的折中方案……

但咱转念一想——这不对啊，如果一个用户一天之内反复访问，看到的推荐文章**一整天都不变**，那和置顶有什么区别？(ノ°Д°)ノ︵┻━┻

## 四、最终方案——Storage 缓存 + 定时刷新

最后在折腾了好几个方案之后，决定把问题分开来处理：

| 问题 | 解决方案 |
|------|---------|
| 页面切换闪白 | 用 `sessionStorage` 缓存上次的推荐结果 |
| 刷新后换一批 | 用 `localStorage` 存时间戳，每隔 6 小时自动换一批 |
| 初始加载太慢 | 后端直接内联第一次的随机结果 |

核心逻辑大概长这样：

```svelte
<script lang="ts">
  import { onMount } from "svelte";

  let { posts, initialPick } = $props();

  let current = $state(initialPick);
  const STORAGE_KEY = "daily-recommend-post";
  const TTL = 6 * 60 * 60 * 1000; // 6小时

  function pickFromCandidates(list) {
    return list[Math.floor(Math.random() * list.length)];
  }

  function rotate() {
    const candidates = posts.filter((p) => !p.data.pinned && p.id !== current?.id);
    if (candidates.length > 0) {
      current = pickFromCandidates(candidates);
    }
  }

  onMount(() => {
    try {
      const cached = JSON.parse(sessionStorage.getItem(STORAGE_KEY) || "null");
      const timestamp = parseInt(localStorage.getItem(STORAGE_KEY + "-ts") || "0");
      const now = Date.now();

      if (cached && now - timestamp < TTL) {
        current = cached;
      } else {
        rotate();
        sessionStorage.setItem(STORAGE_KEY, JSON.stringify(current));
        localStorage.setItem(STORAGE_KEY + "-ts", String(now));
      }
    } catch {
      // storage 不可用时 fallback 到初始值
    }
  });
</script>
```

虽然代码量多了不少，但至少现在：

-  首次加载几乎**零延迟**（后端直接内联）
-  页面切换**不会闪白**（sessionStorage 兜底）
-  每次访问**都有新鲜感**（6小时自动轮换）

## 五、组件的样式打磨

功能搞定了，样式也不能落下。

### 遇到的暗坑

> 💡 **Tip:** 在 Astro 的 Svelte 组件里如果用了 `$props()` 解构，记得要在父组件传值的时候用 `client:load` 而不是 `client:idle`，不然 Swup 页面切换的时候 props 会变成 `undefined`。

这个坑折腾了咱快两个小时……排查到最后发现只是指令用错了，真的想给自己一巴掌。😇
